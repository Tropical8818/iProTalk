# iProTalk — 企业级安全私有化即时通讯系统

iProTalk 是一款专为企业级内部沟通设计的轻量、极速、私有化部署的安全即时通讯（IM）系统。借鉴了开源通讯系统（如 VoceChat）的优秀交互体验，并在此基础上进行了多项性能与安全性升级。

通过将数据资产完全掌控在企业内部服务器上，配合创新的**客户端端到端加密（E2EE）安全机制**，完美解决内部沟通、文件共享、信息同步时的敏感数据泄露隐患。

---

## 🌟 核心设计理念与特色

*   **🔒 绝对的数据隐私（Data Sovereignty）**
    支持 100% 私有化独立部署，所有聊天记录、用户信息、文件数据全部留存在企业内网服务器中，杜绝第三方云服务监控或数据泄露风险。
*   **🔑 创新的端到端加密（E2EE）**
    基于 **Curve25519**、**AES-GCM** 和 **SHA-256** 加密算法。用户密钥完全在浏览器本地生成并以安全方式持久化，服务端仅透传高强度加密密文，从底层实现“纵使服务被破，聊天亦不可破”。端到端加密采用**可选（默认关闭）模式**，兼容企业内部标准与合规审计需求。
*   **⚡ 极简轻量，超低资源占用**
    后端基于 Rust 构建，Poem 异步运行时带来了惊人的高并发性能和极低的内存占用（生产环境冷启动内存 < 20MB），即便在入门级企业虚拟服务器上也能流畅支持数千人同时在线。
*   **💻 现代化的 UI/UX 与实时提醒**
    采用极富科技感的现代化 UI（极简毛玻璃质感、微动效设计），提供即时消息气泡、多端未读红点同步、浏览器后台系统级通知，让协作高效顺畅。

## 🛠 技术栈架构（Technology Stack）

### 📡 后端（Backend Engine）
*   **核心语言**: `Rust 1.85` — 带来无垃圾回收（GC）的高性能与严丝合缝的内存安全保障。
*   **Web 框架**: `Poem` + `Poem-OpenAPI` — 高性能异步多路复用 Web 框架，自动生成强类型端点和 API 文档。
*   **实时通信**: `SSE (Server-Sent Events)` — 极简高效的实时事件单向推送流，告别高开销的轮询和多余的网络协议。
*   **关系型数据库**: `SQLite (via SQLx)` — 关系型基础库，用于存储用户结构、群组属性、频道配置、邀请码关系等，无多余运维开销。
*   **消息日志存储**: `Sled` — 极速的嵌入式 Key-Value 数据库，用于承载高频高并发的消息日志持久化。

### 🎨 前端（Frontend App）
*   **框架**: `React 19` + `TypeScript 5` — 确保高类型安全以及敏捷的渲染更新。
*   **构建工具**: `Vite 7` — 高速打包与极速的热更新体验。
*   **样式表现**: `Vanilla CSS` — 独立原生现代 CSS，完全拒绝 Tailwind 或其他重型第三方框架，通过精心调配的 CSS 变量实现高级的夜间/白天视觉交互。
*   **加密套件**: Web Crypto API (浏览器原生高性能安全硬件调用)，确保本地加解密的高效。

## 📋 功能蓝图（Features）

1.  **即时通讯（IM）**
    *   **频道协作（Channels）**：支持公开频道与私密频道，方便部门快速建立专项协作组。
    *   **直面私信（Direct Messages）**：快速查找企业内所有在职同事并建立 DM 单聊，自动识别对方是否开启 E2EE，动态降级/升级加密安全。
    *   **未读状态计数器**：每个联系人与频道带有实时未读气泡计数，点击进入自动清除并调用后端 `/contacts/update` 同步。
2.  **企业用户目录管理**
    *   **企业全员目录**：支持在单聊面板点击 `+` 呼出企业通讯录，默认展示全员，提供带 300ms 防抖的实时模糊搜索。
    *   **在线状态指示器**：精确展示每个成员的在线/离线状态。
3.  **系统管理中心 (Admin Control Panel)**
    *   **服务器配置**：自由配置系统名称、注册限制条件等。
    *   **邀请码管理**：管理员可生成专属注册邀请码，控制新成员准入权限。
    *   **Webhook 机器人**：为企业现有系统（如 CI/CD、日志警报、运维监控）对接标准 Webhook，轻松向频道推送格式化消息。

## 🚀 部署指南（Deployment）

我们提供了一键式部署套件，支持**原生环境**与 **Docker 容器化**两种高效发布方案。

### 准备工作

克隆并进入项目根目录：
```bash
git clone https://github.com/Tropical8818/iProTalk.git
cd ipro-talk
```

---

### 方案 A：使用一键自动化部署脚本（推荐，原生编译）

我们精心设计了 `deploy.sh` 部署中枢，支持环境检测、前端重构、后端编译、热载运行等全链路管理：

```bash
# 1. 赋予执行权限
chmod +x ./scripts/deploy.sh

# 2. 运行部署向导
./scripts/deploy.sh
```

**向导模式说明**：
*   **选项 1**：全自动安装依赖，构建 React 生产 bundle 并同步资源至 Poem 静态目录，编译 Rust 最终 Release 可执行文件。
*   **选项 2**：通过 Docker Compose 直接在容器中编译并上线。
*   **选项 3**：仅重新编译并覆盖前端 UI 静态资源（适用于前端页面微调后的秒级同步）。
*   **选项 4**：基于现有产物快速启动服务。

---

### 方案 B：使用 Docker Compose 一键启动

若您的服务器支持容器化，可通过标准 Docker 引擎一键挂载持久化存储拉起全套服务：

```bash
# 启动所有服务（后台运行）
docker-compose up --build -d
```

挂载卷卷名与数据留存路径：
*   `./data`：保存 SQLite 关系数据库文件。
*   `./msg_db`：保存 Sled 消息轨迹流。
*   `./backups`：备份文件存放归档目录。

---

## ⚙️ 配置文件说明 (`.env`)

复制环境样例文件 `.env.example` 并编辑 `.env`：

```ini
# 服务端运行端口 (避开 3000 等常用冲突端口)
PORT=3002

# [重要] 用于 JWT 签名的高强度加密密钥
SECRET_KEY=generate_your_strong_secret_key_here

# 数据库路径 (采用 SQLite 数据库)
DATABASE_URL=sqlite:///app/data/iprotalk.db?mode=rwc
MSG_DB_PATH=msg_db

# 日志输出配置
RUST_LOG=poem=info,ipro_talk=info

# 企业第三方 OAuth2 单点登录支持 (可选配置)
GITHUB_CLIENT_ID=your-client-id
GITHUB_CLIENT_SECRET=your-client-secret
GITHUB_REDIRECT_URI=http://localhost:3002/api/auth/oauth2/github/callback
```

## 🧹 备份与容灾机制

为保障企业重要沟通资产的绝对安全，系统在 `scripts/backup.sh` 中集成了安全隔离物理备份套件。您可以通过 Linux `crontab` 计划任务配置每日零点自动热备份：

```bash
# 每天凌晨 2:00 全自动热备份 SQLite 数据库及 Sled 日志流，并自动清除 7 天前的历史副本
0 2 * * * /path/to/ipro-talk/scripts/backup.sh
```
